It is used to load class files automatically on demand.
There must be a pattern on class paths (like class/className.class.php) in order to load it.

Takes a function as parameter.
This child function takes a parameter (the class name) and must have an include (or include_once) command in order to load the class file.

Also, for error handling, it is possible to add a "if (!file_exists(classFile)) { return false; }" before the include_once command.

The loading (child) function will also work for namespaces if the class is instanciated using "new NamespaceName\ClassName()". There must also be a folder inside the class folder which has the same name as the namespace.

In order to detect whether or not the function (child) is being executed from a file on the root folder or the includes folder, one must check if the $_SERVER["HTTP_HOST"] and $_SERVER["REQUEST_URI"] doesn't contain the keyword "includes". If it does, the script must add "../" accordingly.

Finally, one can use the "bruteforce method": scan files inside class directory, using scandir and run a for loop on those files, loading 'em all.
