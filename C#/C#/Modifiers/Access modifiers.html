Access modifiers

One of the pillars of OOP is encapsulation. That means that only the minimum necessary should be visible to external parts of the code.

In this sense, there are access modifiers. They define how visible a property or method should be to the external world.

 

In this sense, properties defined with the private access modifiers should implement getters and setters, which are basically public methods that will set/ return the value of said property on demand. That's where auto-implemented propertiescome in.
