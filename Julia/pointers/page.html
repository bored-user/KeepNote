Just like on JavaScript, Julia stores, in a variable, a pointer to an object in memory.
So, if one sets a variable "x" to  variable "y" which has an object assign to it, Julia won't copy the whole object again. In fact, it'll only create another pointer to that object in memory.
In order to really copy an object, one must use the copy() function.



Example:
a = [1, 2, 3]
b = a
c = copy(a)


a[1] = 'a'
println(a) # ['a', 2, 3]
println(b) # ['a', 2, 3] it's only a pointer to the data in memory, so modifying 'a' modifies 'b' indirectly.

println(c) # [1, 2, 3] it was copied using copy(), that's why it's unchanged.
